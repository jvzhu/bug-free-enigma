# ✅ Bug-Free Enigma — Launch Checklist

Complete day-by-day preparation and launch execution guide.

---

## 🗓 WEEK BEFORE LAUNCH

### Monday (T-7)
- [ ] Finalise all screenshots (6 images, 1270 × 760 px)
- [ ] Create ProductHunt thumbnail (240 × 240 px)
- [ ] Record a 60-second demo GIF or video
- [ ] Review and proofread all copy (tagline, description, first comment)
- [ ] Warm up your ProductHunt account (upvote 5+ products, post a comment)

### Tuesday (T-6)
- [ ] Submit ProductHunt assets (do not schedule yet)
- [ ] Set up email waitlist (Mailchimp / ConvertKit / Buttondown)
- [ ] Add email signup to landing page / GitHub README
- [ ] Brief your support network (friends, colleagues, online communities)
- [ ] Prepare all social media posts (copy from `social-media-content.md`)

### Wednesday (T-5)
- [ ] Create a Buffer / Hootsuite queue with all tweets (scheduled)
- [ ] Draft Reddit posts — have them ready in text files
- [ ] Draft Dev.to article — schedule for launch day morning
- [ ] Prepare HN "Show HN" post
- [ ] Test the live app one more time end-to-end

### Thursday (T-4)
- [ ] Reach out to 5–10 developer friends and ask them to support
- [ ] Post a "coming soon" teaser tweet
- [ ] Engage with ProductHunt community (comment on other products)
- [ ] Confirm your launch date is scheduled in PH
- [ ] Email waitlist: "Launching in 3 days — get ready!"

### Friday (T-3)
- [ ] Final asset review — all images look sharp
- [ ] Test all links (PH link, GitHub, live demo)
- [ ] Write 10 FAQ responses (for PH comments)
- [ ] Prepare upvote request DMs for close contacts
- [ ] Set alarms for 12:01 AM PST launch time

### Saturday (T-2)
- [ ] Finish and queue all email templates
- [ ] Final check: Dev.to article ready, scheduled
- [ ] Backup: have all content saved offline (Google Docs / Notion)
- [ ] Get a good night's sleep plan for launch week

### Sunday (T-1)
- [ ] Confirm ProductHunt submission is scheduled for 12:01 AM PST
- [ ] Confirm all tweets are queued
- [ ] Send "tomorrow is the day!" email to waitlist
- [ ] Final live app test
- [ ] Set your alarm — wake up early on launch day!
- [ ] **Tell 3 people personally:** "Please upvote us tomorrow morning"

---

## 🚀 LAUNCH DAY

### 12:00 AM PST — Pre-launch
- [ ] Check ProductHunt submission is live at 12:01 AM
- [ ] Open dashboard tabs: ProductHunt, GitHub, Twitter, analytics

### 12:01 AM PST — GO LIVE 🚀
- [ ] Confirm product is live on ProductHunt
- [ ] Copy ProductHunt URL and update all scheduled posts
- [ ] Post Maker's First Comment immediately (it's crucial!)
- [ ] Send launch announcement email to waitlist
- [ ] Post Tweet Variant A (launch announcement)
- [ ] DM 5 close friends: "It's live! Please upvote 🙏"

### 6:00 AM PST
- [ ] Reply to every ProductHunt comment received so far
- [ ] Check rank — are you in the top 10?
- [ ] Post to r/webdev (Reddit)
- [ ] Post to r/reactjs (Reddit)
- [ ] Share Dev.to article (if scheduled, confirm it went live)
- [ ] Post Twitter: Feature spotlight (Encryption)

### 8:00 AM PST — Morning momentum
- [ ] Reply to all new PH comments (respond within 30 minutes of posting)
- [ ] Post Twitter: Social proof / metrics tweet
- [ ] Post to r/programming (Reddit)
- [ ] Check if any blog posts or newsletters have picked you up
- [ ] LinkedIn: Post launch announcement

### 10:00 AM PST
- [ ] Post Twitter: Biometrics feature spotlight
- [ ] Reply to Reddit comments
- [ ] Check analytics: where is traffic coming from?
- [ ] Respond to any GitHub issues or stars with a thank you

### 12:00 PM PST — Midday push
- [ ] Post Twitter: Midday ProductHunt re-share
- [ ] LinkedIn: Tag any relevant people in your announcement
- [ ] Post HN "Show HN" (best time: weekday 7–9 AM EST or lunch)
- [ ] Send a message to relevant Slack communities (dev, privacy, open-source)

### 2:00 PM PST
- [ ] Post Twitter: Community engagement question
- [ ] Reply to every HN comment
- [ ] Update waitlist if significant milestone hit (e.g., "We're #3!")

### 4:00 PM PST
- [ ] Post Twitter: Momentum tweet / PH position update
- [ ] LinkedIn: Post 2 (technical deep dive) if not yet posted
- [ ] Screenshot your PH rank for retrospective

### 6:00 PM PST
- [ ] Post Twitter: Thank you midday check-in
- [ ] Reply to all remaining PH comments
- [ ] Personal DMs to anyone who upvoted and connected

### 8:00 PM PST
- [ ] Post Twitter: Final push tweet
- [ ] Send final push email to anyone you know personally
- [ ] Screenshot final PH rank, stats, and comment count

### 11:59 PM PST — End of day
- [ ] Take final screenshots of:
  - ProductHunt position
  - Upvote count
  - Comment count
  - GitHub stars
  - Website analytics
- [ ] Post celebration tweet (if top 5) or thank-you tweet (any position)
- [ ] Write one sentence summary of the day while fresh

## 🛟 CONTINGENCY PLANS

**If rank is below #10 by 8 AM PST:**
- [ ] Send personal DMs to your network asking for support
- [ ] Post in 2–3 Slack/Discord communities
- [ ] Ask 3 friends to share on their Twitter/LinkedIn

**If getting negative comments on PH:**
- [ ] Respond quickly, professionally, and thank them for feedback
- [ ] Don't argue — acknowledge and move on
- [ ] See `producthunt-kit.md` for common objection responses

**If the app has a critical bug on launch day:**
- [ ] Post a PH comment: "We're aware of [issue] and have a fix deploying now"
- [ ] Fix and deploy immediately
- [ ] GitHub Pages auto-deploys on push to main — fast turnaround
