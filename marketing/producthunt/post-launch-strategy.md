## MONTH 2 PLANNING

### Firebase Cloud Sync Launch

**Target:** Week 5–6 post-launch

**Steps:**
1. Set up Firebase project with Firestore
2. Implement client-side encryption before sync (AES-GCM-256)
3. Build cross-device sync UI
4. Beta test with 20 founding members
5. Launch publicly with dedicated ProductHunt post (or update)

**Announcement content:**
- Blog post: "How we built zero-knowledge cloud sync"
- Tweet thread: Technical walkthrough
- Email: "Cloud sync is live for Premium"

### Mobile App Preview

**Target:** Month 2–3

**Steps:**
1. Begin React Native development
2. Share development updates on Twitter (build in public)
3. Open beta sign-up list
4. TestFlight / Play Store beta
5. Full launch

**Content during development:**
- Weekly "Build in public" tweets showing progress
- GitHub repo for mobile app (separate or monorepo)
- Beta user recruitment through existing email list

### Enterprise Features Teaser

**Target:** Month 3+

**Features to announce:**
- Team workspaces
- Admin dashboard
- SSO / SAML integration
- Audit logs export
- Compliance mode (HIPAA, SOC 2 preparation)

**Channels:**
- LinkedIn (primary for enterprise audience)
- Email to users who listed "work" use case
- Hacker News (for the technical/enterprise overlap audience)

### Feature request prioritization framework

For each feature request, score it:

| Factor | Score 1 | Score 2 | Score 3 |
|--------|---------|---------|---------|
| Request volume | 1–3 requests | 4–9 requests | 10+ requests |
| Impact | Nice to have | Useful | Must-have |
| Effort | Days | Weeks | Months |
| Strategic fit | Off-roadmap | Adjacent | Core |

**Priority = (Volume × Impact) / Effort**
